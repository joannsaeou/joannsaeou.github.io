Business use case

A business use case describes "a sequence of actions performed in a business that produces a result of observable value to an individual actor of the business". Hence, from an individual actor’s perspective, a business use case defines the complete workflow that produces the desired results.

Return of Investment(ROI)

ROI stands for "Return of investment" and it is the ratio between the net profit and cost of investman resulting from an investment of some resources. If you want your customer to use API on your open platform, then your API must be pure and simple. You can build up your API to the place it needs to be once you start off at a good rate.

7-step design procedure

Step 1 - List all the parts

The first step is to list all the pieces of data a client application might want to get out of the service or put into it. For example: id, title, dateDue, complete.

Step 2 - Draw State Diagram

You will need to draw out the state diagrams after you reconciled names.

For example : Imagine a box of diagram on your id, title, datedue, and complete. Then draw some arrow pointing to the other box diagram.

Step 3 - Reconcile Magic Strings

Go check on these profile on your API for names you can reuse.

Reusing existing names when possible will lower the chance that a person will misunderstand one of your magic string.

Step 4 - Pick a Media Type

This is where it is time to choose a hypermedia format that can represent the state diagrams. You can design a new domain-specific media type to represent the diagram but it is not necessary. Keep in mind that JSON is not a hypermedia format but, XML is a hypermedia format because you can use Xlink and XForms to give HTML capabiltiies to any XML doucment.

step 5 - Create a profile

your profile can be an ALPS document, a JSON-LD context, or a web page that uses XMDP microformat. You can reuse link relation and descriptors from alps to the original profile.

Step 6 - Write some code (implementation)

Turn over your design documents (state charts and semantic profile) to developers of both the server and client apps in order to start a specific implementation

Step 7 - publish your API

Once everything is running properly including the codes, you are now finish. Your API documentation now consists of your API's billboard URL and a value for the Content-Type header.
