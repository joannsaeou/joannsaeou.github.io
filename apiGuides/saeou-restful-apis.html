What is the RESTful APIs really are?

in RESTFUL term, the API stands for application program interface.

How RESTFUL API are used?

A RESTFUL API is a design that is used to take advantage of existing protocols and HTTP. They are used to allows two software program to communicate with each other.

What are the advantages of REST API?
- Simple to build and adapt
- low uses of resources, meaning the data is not tied to resources or methods.
- process instances are created explicitly
- It provide a great deal of flexibility
What are the disadvantages of REST API?
- REST API can lose the ability to maintain state in REST
- new developers may need to go through a steep of learning curve to notice the disadvantages.
- Cannot retrieve all carrier/class values available for a profile
-Not possible to stage through thousands of products directly
What is the differences between URI and URL

URI stands for uniform resource identifier and URL stands for uniform resource locator

The main difference is that the URI can be a name, locator, or both for an online resource, where the URL is just the locator.

What are Web Communications?

A Web Communication in REST API means that both the HTTP requests used GET/POST/PUT/DELETE as their communications to the client and the web server. You can trigger an HTTP post request where client will enter a message on the box and hit submit. And then, you will receive an HTTP post if the user clicks on the Post button.

API (Application Programming Interface)

When it comes to designing, REST has become the go-to method for designing powerful API that runs over HTTP.

Learning API

For API developers, learning the degree of Restful API design is evaluative if they want high adoption rate for their API.

Four key strategy requirement to API

1. Alignment and Usefulness

2. Engagement and Usability

3. Scalability and Evolvability

4. Manageability and Sustainability

Goals of API design

Enabling Self-service for app developers and app users alike.

Reducing barriers to accessing valuable enterprise resources.

Encouraging collaboration between and among internal and external resources.

Above all, API design must be focused on maximizing the business value of the interface.
