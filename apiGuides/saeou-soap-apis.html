What is SOAP?

SOAP is an application communication protocol,a platform independent, is based on XML, and also a format for sending and inviting messages.

More importantly, SOAP stands for "Simple Object Access Protocol".

Envelope. What are they to SOAP?

An envelope is an element that identifies the XML document as a SOAP message. An envelope is the root element of a SOAP message

Header - What is Header?
- The header is a general tool for adding features to a message in a localized manner without agreement between the communications parties that were involve.
- The header element must be the first immediate child element of ENVELOPE.
- Header defines how a recipient should process the SOAP message.
Faults and Fault Code
- SOAP fault is used to carry any error or status information within a message in SOAP.
- Fault must appear as a body entry but it cannot appear more than once within the body element. So meaning, it must be used ONCE.
- FAULT CODE - it is used by software to include a scientific tool for identifying the fault.
-They must present in a SOAP fault element. and the value must be qualified.
End-to-end data flow of a SOAP API

The data-flow of a SOAP API is that it used the standard HTTP request/response model so that the server uses a "listener" to process the requests.

What are Soap Bindings?

The SOAP conditions defines the formation of the messages, but not how they are swapped.

SOAP - Syntax Rules

MUST be encoded using XML.

- MUST use the SOAP Envelope Namespace

- the messages must not contain XML processing Instructions

SOAP - building blocks

A header element that contains header information

A body element that contains call and response information

A Fault element containing errors and status information
